## 🛠️ ТЕХНОЛОГИЧЕСКИЙ СТЕК

Используй **только** эти технологии. Добавление сторонних библиотек не по списку — ошибка.

### Ядро

| Технология | Назначение |
|---|---|
| **Next.js 14** (App Router) | SSR, SSG, Server Actions, роутинг, оптимизация изображений, metadata API |
| **React 18** | UI-рендеринг |
| **TypeScript** | Строгая типизация по всему проекту |

### Данные и API

| Технология | Назначение |
|---|---|
| **TanStack React Query v5** | Клиентский data fetching — `useQuery` для чтения, `useMutation` для записи |
| **Axios** | HTTP-транспорт |
| **Redux Toolkit** | Клиентское состояние |

### Формы и валидация

| Технология | Назначение |
|---|---|
| **react-hook-form** | Управление состоянием форм |
| **Zod** | Схемная валидация |

### Интернационализация

| Технология | Назначение |
|---|---|
| **next-intl** | i18n — 3 локали (`ru`, `kk`, `en`), роутинг через middleware |

### UI и стили

| Технология | Назначение |
|---|---|
| **bcc-design** | Внутренняя библиотека компонентов BCC (кнопки, модалки, формы, скелетоны и т.д.) |
| **bcc-design-icons** | Внутренняя библиотека иконок |
| **bcc-styles** | Базовые стили BCC |
| **Sass/SCSS + CSS Modules** | Стили компонентов |
| **clsx** | Условное объединение классов |
| **framer-motion** | Анимации |
| **@splidejs/react-splide** | Карусели / слайдеры |
| **lenis** | Плавный скролл |

> ⚠️ **Обязательно:** Все UI-компоненты (кнопки, инпуты, модалки, иконки и т.д.) должны использоваться **только из `bcc-design`, `bcc-design-icons` и `bcc-styles`**. Создавать собственные аналоги этих компонентов запрещено.

### Качество кода

| Технология | Назначение |
|---|---|
| **ESLint** (Airbnb config) + **Prettier** | Единообразие кода |
| **Husky + lint-staged** | Pre-commit хуки |

## 🏛️ АРХИТЕКТУРА — Feature-Sliced Design (FSD)

Структура проекта строго следует FSD:

```
src/
├── app/              ← Next.js App Router: layout, route pages, providers, глобальные стили
├── screens/          ← Компоновки страниц (About, Main, Vacancies и т.д.)
├── widgets/          ← Переиспользуемые блоки из нескольких фич (Navbar, Footer)
├── features/         ← Изолированные бизнес-фичи
├── entities/         ← Доменные сущности
└── shared/           ← Утилиты, API-клиенты, хуки, константы, UI-примитивы
```

**Псевдоним пути:** `@/` → `src/`

## 🤖 Как работать с AI-агентом

### Общий подход:

Работай с AI итеративно — короткими циклами «промпт → результат → проверка → уточнение».

### Инициализация проекта:

```
Создай полный boilerplate для Next.js 14 App Router проекта.
Стек: [перечисли из раздела выше].
Архитектура: FSD (src/app, src/screens, src/widgets, src/features, src/entities, src/shared).
Добавь базовые провайдеры, middleware для next-intl, конфиги ESLint и Prettier.
```

### Вёрстка по Figma:

```
Ты — Senior Frontend Developer. Верстаешь по Figma-макету.

## Стек
- Next.js 14 App Router + TypeScript strict mode
- FSD архитектура
- SCSS Modules (не Tailwind, не inline styles)
- Path alias: @/* → ./src/*

## Правила FSD
- src/widgets/ — только компоненты, используемые на нескольких страницах (Navbar, Footer)
- src/screens/<page>/ui/ — все секции, уникальные для страницы
- src/shared/ — токены, утилиты, store, UI-примитивы

## Правила CSS
- Mobile-first: базовые стили = 375px, десктоп = @media (min-width: 1280px)
- Все CSS-переменные в src/app/styles/globals.scss под :root {}
- height: Xpx (точно) для фиксированных Figma-фреймов, не min-height

## Figma → Код
1. Получи контекст и скриншот для десктопного узла, затем мобильного
2. Внимательно читай Figma-данные перед написанием CSS
3. Скачай все ассеты сразу — ссылки из Figma MCP истекают через 7 дней
```

### Уточняй детали:

- Шрифты и их веса
- Цвета (основные, акцентные, фоны)
- Отступы и выравнивание
- Анимации (framer-motion)
- Адаптивные breakpoints

## 💡 СОВЕТЫ ОТ ОПЫТНОГО РАЗРАБОТЧИКА

✨ **Как подходить к такому заданию с AI:**

1. **Начни с инициализации:**
   - Дай AI детальный промпт со всем стеком и архитектурой
   - Проверь что структура папок соответствует FSD
   - Убедись что провайдеры и конфиги работают (`npm run dev` без ошибок)

2. **Затем вёрстка:**
   - Загрузи Figma-ссылку или скриншоты в AI
   - Верстай секцию за секцией, не всё сразу
   - Проверяй каждую секцию в браузере перед переходом к следующей

3. **Потом функциональность:**
   - Описывай каждую фичу изолированно
   - Следуй FSD — каждая фича в своей папке `src/features/<feature-name>/`
   - Используй React Query для запросов, Redux для глобального стейта

4. **Отлаживай с AI:**
   - Копируй ошибки из консоли или TypeScript прямо в промпт
   - Спрашивай почему что-то не совпадает с макетом
   - Проси объяснить решение, не только код

5. **Итоговая проверка:**
   - Прогони весь чеклист перед push
   - Проверь production build (`npm run build && npm run start`)
   - Протестируй все три локали
